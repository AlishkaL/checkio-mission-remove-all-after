Da Sie am Ende i-s nach dem border nehmen müssen, können Sie ohne Zweifel einfach jedes i in result einfügen. Verwenden Sie dazu die Methode list.append(). 

result = []
for i in items:
    result.append(i)

Wie führe ich die Überprüfung der Werte und die Unterbrechung der Iteration durch, wenn ich bereits border in result eingefügt habe?

So einfach ist das, wie Sie sagten:) Sie haben bereits die aktuelle i in result eingefügt. Prüfen Sie nun mit der Anweisung if, ob i mit dem Operator == gleich border ist. Wenn ja, verwenden Sie den Befehl break, um die Iteration zu beenden. 

result = []
for i in items:
    result.append(i)
    if i == border:
        break
...

Wie sollte also meine endgültige Lösung aussehen?

Das ist fast alles. Sie müssen Ihren Code nur in einen Funktionsausdruck verpacken und die Liste zurückgeben, die Sie gefüllt haben - result.

Hier ist die gesamte Lösung:

from collections.abc import Iterable


def remove_all_after(items: list[int], border: int) -> Iterable[int]:

    result = []
    for i in items:
        result.append(i)
        if i == border:
            break
            
    return result

Sie können versuchen, den obigen Code zu "optimieren" und return result in die Anweisung if anstelle von break einzufügen. Aber wenn border nicht in items enthalten ist oder items überhaupt leer ist, gibt Ihre Funktion nichts zurück (None)!

Diese Lösung sieht ein bisschen aus wie die vorherige, verwendet aber die Funktion generator. Sie gibt jeden Wert "außerhalb" der Funktion aus und hält an, wenn border der letzte ausgegebene Wert war. 

from collections.abc import Iterable


def remove_all_after(items: list[int], border: int) -> Iterable[int]:

    for i in items:
        yield i
        if i == border:
            break

Bei dieser Lösung wird ein Teil von items einschließlich border zurückgegeben, nachdem dessen Index oder der gesamte items gefunden wurde. 

from collections.abc import Iterable


def remove_all_after(items: list[int], border: int) -> Iterable[int]:

    return items[:items.index(border)+1] if border in items else items

Warum hat die Methode list.index() hier keine Ausnahme ausgelöst? Weil if border in items trotz der geschriebenen Reihenfolge der Befehle tatsächlich zuerst ausgeführt wird!
